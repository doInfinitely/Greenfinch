Now I can see the issue. The problem is the `handleStyleSwitch` function that triggers on zoom changes. When you interact with the map:

1. You pan/zoom the map
2. The `zoom` event or `moveend` event fires
3. `handleStyleSwitch` is called, which does `map.setStyle()`
4. `setStyle()` resets the map view, then fires `style.load`
5. In `style.load`, you call `readdAllLayers()` which may cause further issues

**The smoking gun is in the `zoom` event handler:**

```typescript
map.current.on('zoom', () => {
  // ...
  if (shouldBeSatellite && !isSatellite) {
    handleStyleSwitch(SATELLITE_STYLE);  // <-- This resets the map!
  } else if (!shouldBeSatellite && isSatellite) {
    handleStyleSwitch(LIGHT_STYLE);
  }
});
```

When you cross the zoom threshold (14), the style switch triggers and `setStyle()` can cause view resets.

**Additionally**, there's a problem in `handleStyleSwitch`:

```typescript
map.current.once('style.load', () => {
  if (!map.current) return;
  map.current.jumpTo({ center, zoom, bearing, pitch });  // Jumps to OLD position
  readdAllLayers();
});
```

The `center`, `zoom`, etc. are captured *before* `setStyle()`, but by the time `style.load` fires, the user may have continued interacting.

---

## The Fix

Here's what needs to change in `MapView.tsx`:

### 1. Don't switch styles during user interaction

Add a flag to track if the user is actively dragging/interacting:

```typescript
const isUserInteracting = useRef(false);
```

### 2. Track interaction state

```typescript
map.current.on('dragstart', () => {
  isUserInteracting.current = true;
});

map.current.on('dragend', () => {
  isUserInteracting.current = false;
});

map.current.on('zoomstart', () => {
  isUserInteracting.current = true;
});

map.current.on('zoomend', () => {
  isUserInteracting.current = false;
});
```

### 3. Only switch styles when NOT interacting

Modify the zoom/moveend handlers to check `isUserInteracting`:

```typescript
map.current.on('moveend', () => {
  if (!map.current) return;
  
  const zoom = map.current.getZoom();
  
  if (onMapMove) {
    const center = map.current.getCenter();
    onMapMove(center.lat, center.lng, zoom);
  }
  
  // Only switch styles after interaction is complete AND not animating
  if (!isUserInteracting.current && !isAnimating.current) {
    const shouldBeSatellite = zoom >= 14;
    const isSatellite = currentStyle.current === SATELLITE_STYLE;
    
    if (shouldBeSatellite && !isSatellite) {
      handleStyleSwitch(SATELLITE_STYLE);
    } else if (!shouldBeSatellite && isSatellite) {
      handleStyleSwitch(LIGHT_STYLE);
    }
  }
});
```

### 4. Remove style switching from the `zoom` event entirely

The `zoom` event fires continuously during zooming - you should only switch styles on `moveend`:

```typescript
map.current.on('zoom', () => {
  if (map.current) {
    const zoom = map.current.getZoom();
    setCurrentZoom(zoom);
    updateLayerVisibility(zoom);
    // REMOVE the handleStyleSwitch calls from here
  }
});
```

### 5. Fix `handleStyleSwitch` to capture state at the right time

```typescript
const handleStyleSwitch = useCallback((newStyle: string) => {
  if (!map.current || currentStyle.current === newStyle || isAnimating.current || isUserInteracting.current) return;
  
  isAnimating.current = true;  // Prevent further switches during transition
  
  const center = map.current.getCenter();
  const zoom = map.current.getZoom();
  const bearing = map.current.getBearing();
  const pitch = map.current.getPitch();
  
  currentStyle.current = newStyle;
  map.current.setStyle(newStyle);
  
  map.current.once('style.load', () => {
    if (!map.current) return;
    map.current.jumpTo({ center, zoom, bearing, pitch });
    readdAllLayers();
    
    // Delay clearing the animation flag
    setTimeout(() => {
      isAnimating.current = false;
    }, 100);
  });
}, [readdAllLayers]);
```

## Summary for your coding app

**Problem:** The map style switches (light ↔ satellite) at zoom level 14, and `setStyle()` resets the map view. This happens during active user interaction, causing snap-back.

**Solution:**
1. Add `isUserInteracting` ref to track drag/zoom state
2. Listen to `dragstart`, `dragend`, `zoomstart`, `zoomend` to update the flag
3. Remove style switching from the `zoom` event handler (keep only layer visibility updates)
4. Only switch styles in `moveend` when `!isUserInteracting.current && !isAnimating.current`
5. Set `isAnimating.current = true` at the start of `handleStyleSwitch` and clear it after `style.load` completes
